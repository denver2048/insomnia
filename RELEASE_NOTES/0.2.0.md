# Release Notes

## v0.2.0 (2025-03-12)

Helm-based deployment, streamlined provisioning (infracore + app via Helm), OpenAI secret support, and improved logging.

### Features

- **Helm chart** — Insomnia is deployed via Helm from `charts/insomnia/` (Deployment, Service, ServiceAccount, RBAC). Values for image, Prometheus/Loki URLs, and optional OpenAI secret name.
- **Root provision script** — Single entrypoint `./provision.sh` at repo root: runs infracore (Kind + Prometheus stack + alert rules), builds the app image, loads it into Kind, and runs `helm install` for Insomnia. If `OPENAI_API_KEY` is set in the environment, creates the Kubernetes secret and restarts the deployment.
- **Infracore-only provisioning** — `infracore/provision.sh` now provisions only infrastructure (Kind cluster, kube-prometheus-stack, Alertmanager config, Prometheus alert rules). Insomnia app is no longer deployed from infracore; use root `provision.sh` for the full stack.
- **OpenAI secret** — Chart expects optional secret `insomnia-openai` with `OPENAI_API_KEY` (and optionally `OPENAI_MODEL`) injected via `envFrom`. Script `scripts/create-openai-secret.sh` creates/updates the secret; root provision creates it when `OPENAI_API_KEY` is set.
- **LLM module** — `llm.py` moved to `agent/llm.py`. Uses OpenAI when `OPENAI_API_KEY` is set; otherwise returns a stub report and logs an error (startup check + per-request log and stderr message) so missing key is visible in pod logs.
- **Logging** — Uvicorn access log no longer records GET `/healthz` and GET `/readyz`. Step-by-step pipeline logging: alert received, pipeline start/finish, each investigator (kubernetes, logs, metrics, registry), root-cause analysis, and full LLM answer in pod logs.
- **Alert webhook** — Alert payload handling supports both `alerts` (list) and `alert` (single); first alert is used when list is provided (e.g. Alertmanager). Response `{"status": "processed"}` when investigation completes.

### Infrastructure & tooling

- **charts/insomnia/** — Helm chart (Chart.yaml, values.yaml, templates: deployment, service, rbac, sa, optional openai-secret from values).
- **provision.sh** (root) — Full stack: infracore → build image → load into Kind → Helm install; optional OpenAI secret from `OPENAI_API_KEY`/`OPENAI_MODEL`.
- **infracore/provision.sh** — Infra only: Kind, kube-prometheus-stack, Alertmanager config, alert rules (no Insomnia app).
- **scripts/create-openai-secret.sh** — Creates/updates `insomnia-openai` secret in `insomnia` namespace from `OPENAI_API_KEY` and optional `OPENAI_MODEL`.
- **AlertmanagerConfig** — Removed unsupported `send_resolved` field for v1alpha1 compatibility.
- **.gitattributes** — `*.sh text eol=lf` to avoid CRLF-related “bash\r: No such file or directory” on run.

### Configuration

- **Environment** — App reads `OPENAI_API_KEY` and `OPENAI_MODEL` from the optional secret `insomnia-openai` (or env). Missing key is logged at startup and when analysis runs.
- **Helm** — `openai.existingSecret` (default `insomnia-openai`), optional `openai.createSecret` + `openai.apiKey`/`openai.model` to create the secret from values.

### Dependencies

- Same as 0.1.0; `openai>=1.0.0` added in `requirements.txt` at repo root. App listens on port 8000 (Dockerfile/Helm).
