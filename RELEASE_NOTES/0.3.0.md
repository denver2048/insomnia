# Release Notes

## v0.3.0 (2025-03-15)

AI system stack: AgentGateway and Kubernetes Gateway API for LLM routing, Kagent with built-in agents disabled, and Insomnia as the single agent using the gateway for root-cause analysis.

### Features

- **AI system Helm chart** — New `charts/ai-system/` deploys AgentGateway (control plane + proxy), Gateway API resources (Gateway, HTTPRoute, AgentgatewayBackend), and optional Kagent ModelConfig. OpenAI backend (default model gpt-5.2) with API key from a Kubernetes Secret; no hardcoded credentials.
- **AgentGateway (kgateway)** — Central LLM gateway: exposes OpenAI-compatible `/v1` endpoint, routes traffic to the configured backend, and applies auth from a Secret. Proxy logging configurable via AgentgatewayParameters (level, format) so kgateway activity is visible in logs.
- **Kagent with built-in agents disabled** — Provision script installs Kagent (controller + CRDs) with all default agents turned off (`agents.*.enabled=false`). No helm-agent, observability-agent, istio-agent, etc.; Insomnia is the only agent that uses the LLM.
- **Insomnia as agent via gateway** — Insomnia sends LLM requests through AgentGateway instead of directly to OpenAI. New env `OPENAI_BASE_URL` (and Helm value `openai.baseUrl`) points the app at the gateway; root `provision.sh` sets it to `http://agentgateway-proxy.agentgateway-system.svc.cluster.local/v1` after ai-system is deployed. Flow: Insomnia → Gateway → AgentgatewayBackend → OpenAI.
- **Root provisioning includes ai-system** — `./provision.sh` now runs ai-system provisioning (Gateway API CRDs, AgentGateway, Kagent CRDs, ai-system chart, Kagent with agents disabled) and then configures Insomnia to use the gateway.

### Infrastructure & tooling

- **charts/ai-system/** — Helm chart: Gateway, AgentgatewayParameters (logging), AgentgatewayBackend (OpenAI), HTTPRoute, ConfigMaps (gateway, backend, model-routing), optional Kagent ModelConfig. Values for namespace, AgentGateway system namespace, LLM provider/model/secret, Kagent gateway URL, and logging (level/format).
- **scripts/provision-ai-system.sh** — Installs Gateway API CRDs (v1.5.0), AgentGateway CRDs + control plane (with `controller.logLevel=debug`), OpenAI secret in `agentgateway-system`, Kagent CRDs (before chart so ModelConfig applies), ai-system chart, then Kagent with all built-in agents disabled.
- **provision.sh** (root) — New steps: run ai-system script, then configure Insomnia for gateway (`helm upgrade` with `openai.baseUrl` and rollout restart).
- **charts/insomnia/** — New `openai.baseUrl` value; when set, deployment gets `OPENAI_BASE_URL` so the app uses the gateway as the LLM endpoint.
- **agent/llm.py** — Uses `OPENAI_BASE_URL` when set: `OpenAI(api_key=..., base_url=...)` so all chat completions go to the gateway.

### Configuration

- **AgentGateway** — Secret `openai-secret` in `agentgateway-system` (key `Authorization`). AgentgatewayParameters `agentgateway-proxy-params` sets proxy logging (e.g. `level: debug`, `format: json`). Gateway references it via `spec.infrastructure.parametersRef`.
- **Insomnia** — Optional `openai.baseUrl` in Helm values (e.g. gateway URL); root provision sets it when ai-system is deployed. Existing `openai.existingSecret` (e.g. `insomnia-openai`) still supplies `OPENAI_API_KEY` for the client; gateway injects its own key for upstream.
- **Kagent** — Installed in `ai-system` namespace with `providers.default=openAI` and every `agents.<name>.enabled=false`.

### Fixes & operations

- **Kagent CRDs before ai-system chart** — Provision order changed so Kagent CRDs are installed before the ai-system Helm chart; avoids “no matches for kind ModelConfig” when the chart creates the ModelConfig resource.
- **Namespace ownership** — Removed `templates/namespace.yaml` from the ai-system chart so Helm does not try to adopt an existing `ai-system` namespace that lacked Helm labels/annotations.
- **Shell script line endings** — `scripts/provision-ai-system.sh` and `scripts/run-builtin-agent.sh` corrected from CRLF to LF to fix “env: bash\r: No such file or directory”.

### Documentation

- **README.md** — AI system section updated: Kagent built-in agents disabled, Insomnia as the agent, flow Insomnia → Gateway → OpenAI. Agent section now describes triggering alerts and using demo-cases rather than kagent invoke.
- **charts/ai-system/README.md** — Architecture and configuration; notes that Insomnia is the agent and built-in agents are disabled.
- **charts/ai-system/NOTES.txt** — Post-install steps include viewing kgateway proxy logs and configuring Insomnia as the agent.

### Dependencies

- Unchanged from 0.2.0 for the Insomnia app. Ai-system provisioning requires `kubectl`, `helm`; optional `kagent` CLI for debugging (built-in agents are not used).
