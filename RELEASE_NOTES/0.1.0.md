# Release Notes

## v0.1.0 (2025-03-10)

Initial release of **Insomnia** — Intelligent Night Operations Monitoring & Investigation Agent: an AI-powered Kubernetes incident analysis service.

### Features

- **Alert webhook ingestion** — FastAPI service on port 8080 that receives Prometheus/Alertmanager webhooks via `POST /alert`. Parses alert labels (namespace, pod) and triggers the investigation pipeline.
- **LangGraph investigation pipeline** — State graph that runs investigators in sequence:
  - **Kubernetes investigator** — Fetches pod details and events from the cluster via MCP (`pods_get`, `events_list`).
  - **Log investigator** — Fetches pod logs via MCP (`pods_log`).
  - **Evidence aggregation** — Builds a single evidence payload scoped to the alerted pod.
  - **Root-cause analysis** — Sends evidence to an LLM (OpenAI) for structured report generation.
- **Image incident enrichment** — For image-related alerts (e.g. ImagePullBackOff):
  - **ECR** — Detects ECR images and checks `imagePullSecrets` for ECR auth.
  - **Docker Hub** — Queries tag availability for Docker Hub images.
  - **Other registries** — Flags non-ECR/non-Docker-Hub registries for manual review.
- **Structured LLM reports** — SRE-style output: Root Cause, Confidence, Evidence, Suggested Fix (configurable via `OPENAI_MODEL` and `OPENAI_API_KEY`).
- **MCP integration** — SSE-based MCP client for a Kubernetes MCP server (default `http://localhost:8081/sse`) exposing cluster read operations.

### Infrastructure & demo

- **infracore/** — Prometheus/Alertmanager and RBAC for Insomnia:
  - `alert-rules.yaml` — PrometheusRule for ImagePullBackOff, PodPending, OOMKilled with `insomnia: "true"` label.
  - `alertmanager-config.yaml` — AlertmanagerConfig routing matching alerts to the Insomnia webhook.
  - `rbac.yaml` — ClusterRole/Binding for read-only MCP server access (pods, events, logs).
- **provision.sh** — One-shot local provisioning: Kind cluster, kube-prometheus-stack, Kubernetes MCP Server (Helm), Alertmanager config, alert rules, and RBAC.
- **demo-cases/** — Example manifests to trigger alerts:
  - `OOM.yaml` — OOMKilled (memory limit + memory-hog).
  - `Pending.yaml` — Pending (unschedulable resource requests).
  - `ImagePull-tag.yaml` — ImagePullBackOff (missing image tag).

### Configuration

- **Environment** — `OPENAI_API_KEY`, `OPENAI_MODEL` (e.g. `gpt-4o`); see `insomnia/.env.example`.
- **MCP server** — Expected at `http://localhost:8081/sse` with tools: `pods_get`, `events_list`, `pods_log`.

### Dependencies

- FastAPI, Uvicorn, LangGraph, OpenAI, python-dotenv, requests.
